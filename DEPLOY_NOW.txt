## STEP 3: ADD MYSQL DATABASE (1 minute)

4. Click **"Deploy"**

Railway creates database and credentials automatically.

✅ **Done with Step 3 - Let it deploy (2 minutes)**

---

## STEP 4: SET ENVIRONMENT VARIABLES (2 minutes)

In Railway Dashboard:
1. Look for a **"Variables"** tab on the left
2. Click to open Variables section
3. Copy this entire text block:

```
APP_ENV=production
APP_DEBUG=false
APP_URL=https://${{ RAILWAY_PUBLIC_DOMAIN }}
DB_HOST=${{ MYSQL.PGHOST }}
DB_PORT=${{ MYSQL.PGPORT }}
DB_NAME=railway
DB_USER=${{ MYSQL.PGUSER }}
DB_PASSWORD=${{ MYSQL.PGPASSWORD }}
CSRF_TOKEN_LIFETIME=3600
RATE_LIMIT_LOGIN_ATTEMPTS=5
RATE_LIMIT_RESET_ATTEMPTS=3
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your-email@gmail.com
EMAIL_PASSWORD=your-app-password
EMAIL_FROM=noreply@ideaspace.com
SESSION_TIMEOUT=3600
SESSION_SECURE=true
SESSION_HTTPONLY=true
MAX_UPLOAD_SIZE=5242880
LOG_LEVEL=error
```

4. Paste into Railway variables
5. **Replace these two lines ONLY:**
   - `EMAIL_USER=your-email@gmail.com` → Use your real Gmail
   - `EMAIL_PASSWORD=your-app-password` → Use Gmail app password

**How to get Gmail app password:**
- Go: https://myaccount.google.com/apppasswords
- Select "Mail" + "Windows Computer"
- Generate password
- Copy the 16-character password

6. Click **"Save"** or **"Deploy"**

✅ **Done! Your app is live!**

## ⚡ FINAL STEP: RUN DATABASE MIGRATIONS (2 minutes)

All the database tables are created automatically, BUT run this command to make sure everything is complete:

**On your computer, open terminal and run:**

```bash
cd /workspaces/Ideaspace
railway login
```

You'll be asked to authorize. Click the link and authorize.

Then run:
```bash
railway link
```

Select your Ideaspace project.

Then run:
```bash
railway shell
php migrate.php
exit
```

This sets up all 25+ database tables.

✅ **Done!**

## 🆘 TROUBLESHOOTING

### "App won't load"
- Wait 3-5 minutes for initial deployment
- Refresh page (Ctrl+R or Cmd+R)
- Check Railway dashboard → Logs for errors

### "Database connection failed"
- Make sure MySQL service is Running in Railway
- Set environment variables correctly
- Run: `railway shell` → `php migrate.php`

### "Email not sending"
- Verify Gmail app password is correct
- Use app-generated password from: https://myaccount.google.com/apppasswords
- Not your regular Gmail password!

### "Something is broken"
- Check Railway logs: Dashboard → Logs
- Look for PHP errors
- Verify all environment variables are set
